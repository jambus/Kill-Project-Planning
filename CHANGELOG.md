# Changelog

## [1.0.1] - 2026-04-30

### 新特性 (New Features)
- **AI 优先级微批次排期与完整性回滚 (Priority Mini-Batches & Integrity Rollback)**：
  - 弃用容易产生“先到先得”次优解的一把梭分配。现在系统严格按照 CSV 导入的高低优先级，将项目划分为“小批次”发送给大模型处理。
  - **排期完整性审计 (Integrity Audit)**：在所有批次排期完成后，系统会执行事务级检查，自动剥离并回滚只排上开发没排上测试的“半拉子工程”，将闲置资源退回。
- **AI 批量全局排期与 Prompt Caching (Global Batch Scheduling)**：通过合并单次调用的上下文并依托大模型的 Prompt Caching 机制，为批量分配节省了 90% 以上的 API Token 消耗。
- **多阶段迭代排期架构 (Two-Phase Scheduling)**：将排期阶段分离为 Dev-first 和 Test-second，优先安排开发任务。
- **测试前置依赖双重约束**：动态计算项目所有开发任务的起止时间中点作为测试最早开始时间，同时在 AI Prompt 中加入强制校验规则（确保测试不可早于开发），防止测试人员被盲目锁定。
- **AI 排期准则完全可配置 (Editable AI Prompt)**：在“系统设置”页面提供了一个可编辑的多行文本框，暴露了 AI 排期引擎的核心调度 Prompt。允许高级用户直接向大模型追加自定义的排期准则与限制，并提供了一键恢复系统默认规则的按钮。
- **排期策略动态切换 (Strategy Templates)**：新增“专注模式(100%投入)”、“均衡模式(50%分时并行)”与“紧急模式(超负荷推进)”等三种排期策略。
- **节假日与日历可配置 (Configurable Holiday Calendar)**：新增系统“节假日管理”页面，支持自定义调整全局法定假日和调休工作日，避免跨年问题。
- **严格排期时间窗口与 Token 优化 (Strict Window & Early Exit)**：
  - 系统强制以用户选择的结束月份为绝对物理边界（如选 4-6月则 6月30日为死线），超过此边界的项目需求将被截断为未排缺口。
  - **引擎熔断优化**：若某位工程师档期已被排满或越界，AI 引擎会自动停止在后续项目中询问该工程师；当所有工程师资源均被分配完毕时，系统将直接触发早期退出（Early Exit）中止后续循环请求，大幅节省 API Token 和执行时间。
- **一键下载导入模板 (Sample CSV Download)**：在项目管理页面提供带有标准多维表头及脱敏示例数据的 CSV 模板文件下载。
- **排期进度与错误反馈增强**：
  - 增加分布状态面板，实时反馈「初稿->审计->补排」的流转状态。
  - 优化错误弹窗提示（AI Key 过期、网络报错等）及一键跳转配置功能。

### 优化与修复 (Improvements & Fixes)
- **杜绝“0天排期”问题**：底层强依赖代码进行 `Math.min` 硬扣减截断计算，并增设严格的 $\ge$ 1 整数人天过滤逻辑。
- **重构全栈工程师职责**：将“全栈工程师”的默认匹配严格限定在开发（Dev）阶段（禁止其代替测试），以保证核心测试人员利用率。
- **增强文件导入兼容性 (Enhanced File Import)**：改用智能表头模糊匹配替代按列号读取，支持识别 `Tech Lead`、`Quality Lead`、`Details Product MD`、`Tech Stack` 及 `Domain` 等多个扩展业务字段，并在数据大盘中结构化展示。
- **项目过滤与分离展示**：大盘中自动隔离 `devTotalMd` 和 `testTotalMd` 均为 0 的项目，放置在底部“待评估项目”清单中，不参与 AI 排期。
- **已排期任务分组视图**：大盘支持「按人员分组」与「按项目分组」双向切换，方便多维统计查看。
- **人员管理体验优化**：修复人员负荷滑动条（Range Input）因步长限制无法精确设置为 0% 或 100% 的边界值问题。
- **时间合法性容错校验**：增强了日期有效性校验，修复了因导入脏数据导致的 `RangeError: Invalid time value`。
- **精简 Chrome 权限**：移除了未使用的 `alarms` 权限，以满足 Chrome Web Store 审核的安全规范。
- **完善 Chrome Web Store 上架素材**：生成了全新的 SVG & PNG (`16/48/128px`) 品牌 Logo 并引入自动化打包与发布命令 (`npm run publish`)。

## [1.0.0] - 2026-04-26

### 初版发布 (Initial Release)
- **核心排期引擎上线**：发布基于纯客户端 (Local-first) 架构的 Intelligent Resource Planner (IRP) 首个正式版本，所有数据缓存在浏览器本地 IndexedDB，彻底保障数据隐私。
- **项目数据接入**：
  - 支持通过上传 CSV 或 Excel (.xlsx) 文件，一键批量导入待排期的项目清单、优先级和预估的研发/测试工时。
  - 项目优先级严格遵循导入文件从上到下的物理顺序，确保符合现实管理直觉。
- **人员与资源管理**：
  - 提供可视化的人员管理面板，支持录入团队成员的专业角色（前端/后端/全栈/APP/测试）、当前可用负荷比例（1-100%）及各类技术栈技能标签。
- **AI 智能排期大盘 (Dashboard)**：
  - 接入大语言模型 (支持 OpenAI 及各类兼容的本地/云端 LLM 接口)，基于全局待排项目和可用资源池一键生成智能排班建议。
  - 独创的双向核算审计看板：直观展示每个项目的剩余人天需求（资源缺口），以及每个团队成员的当前闲置时长（空置资源）。
  - 根据分配占比和项目周期，自动推算跨月份的有效投入人天（MD）。
- **Jira 深度集成与原生级预警 (Content Script)**：
  - 插件支持在后台异步关联 Jira 平台的工作动态。
  - 当项目经理浏览 Jira 的 Issue 页面时，系统会实时读取该 Issue 负责人的本地排期库负荷，并在页面 DOM 内无侵入地悬浮注入其“红黄绿”过载状态，提供风险干预预警。
- **系统底层与合规**：
  - 完整的选项配置页面，支持自定义配置大模型 API 域名、密钥与指定模型。敏感密钥使用 Chrome Storage 隔离加密存储。
  - 标准化开发文档、打包流程、隐私合规声明 (Privacy Policy) 以及 SVG 品牌规范已初步建立。
