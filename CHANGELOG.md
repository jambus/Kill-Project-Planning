# Changelog

## [1.0.3] - 2026-05-17

### 新特性 (New Features)
- **产品运维管理 (Product Operations Management)**：
  - **独立配置与管理模块**：新增了专门的产品运维配置页面，支持按产品名称配置每月的固定开发与测试运维人天（MD），并提供完整的增删改查及 CSV 批量导入/导出能力。
  - **技能字典联动生成**：无论是通过界面手动配置还是批量导入新的产品运维项，系统都会自动将其产品名称加入到全公司的「技能管理」字典中（标记为业务技能），极大地方便了后续对人员进行技能打标。
  - **PASS 0 按月基石排期算法**：排期引擎新增「阶段零」前置调度逻辑。在 AI 处理高优先级业务项目前，系统优先通过纯代码逻辑强制扣减运维人天。**系统采用精细的按月 (Month-by-month) 独立遍历机制，强制分配落在当月物理边界内**，彻底消除了多月运维人天在首月扎堆堆积的问题。
  - **智能好资源保护策略**：排期算法能够智能识别当前批次所有待排项目的“技术负责”与“测试负责”，在分配日常运维任务时，会自动避开这些核心 Leader（好资源），优先分派给仅拥有该产品技能标签的“基本资源”，实现人才效能最大化。
  - **大盘可视化标识**：在全局排期大盘中，自动为运维任务生成的虚拟分配记录（以绿色 `[运维] Product Name` 标识），使运维占用与常规项目一目了然。

### 修复与稳定性 (Fixes & Stability)
- **数据持久化重磅修复 (Critical Persistence Fix)**：
  - **修正 Dexie Schema 版本演进逻辑**：修复了在数据库版本升级过程中，因后续版本未完全定义所有数据表而导致部分表（如 `projects`, `resources`, `settings`）被意外删除的严重 Bug。现在所有版本定义均显式包含全量表结构，确保数据在 schema 变更时完美继承。
  - **稳定扩展 ID (Stable Extension ID)**：在 `manifest.json` 中配置了固定的 `key`（基于项目私钥提取）。解决了在开发环境下因 ID 随机变化导致 `chrome.storage.local` 和 IndexedDB 存储原点（Origin）重置的问题，保障了系统设置与业务数据的长期持久化。
- **统一报错反馈机制 (Unified Error Feedback)**：
  - **抽象化通用报错组件 (ErrorModal)**：封装了标准化的全局报错弹出窗口组件，替代了原始的 `window.alert()`。
  - **全量 UI 覆盖**：在「项目管理」、「人员管理」及「技能管理」等所有数据导入入口集成了该组件。当文件格式非法或解析失败时，系统将以更加专业、友好的图层模态框展示错误详情。
  - **日志追踪增强**：所有业务报错同步通过 `console.error()` 输出至浏览器控制台，并包含详细的技术上下文，极大提升了排障效率。
- **导入鲁棒性大幅提升 (Import Robustness)**：
  - **修复 `undefined` 报错**：修复了在导入项目、人员或技能时，因部分行数据缺失导致 `.includes()` 或 `.toString()` 调用崩溃的 Bug。
  - **修复中文乱码**：通过显式指定 UTF-8 编码提示，修复了部分不带 BOM 的 CSV 文件在导入时出现中文乱码（如 `çº¿ä¸`）的问题。
  - **支持忽略多余字段**：优化了列匹配逻辑，系统现在能更智能地识别核心表头并自动忽略文件中的无关冗余字段，增强了对不同格式 CSV/Excel 的兼容性。
### 优化与增强 (Improvements & Enhancements)
- **调度引擎核心性能优化 (Scheduling Engine Optimization)**：
  - **增量矩阵更新 (Incremental Matrix Update)**：重构了资源日历槽位矩阵的管理方式，引入内存共享的 `DailySlot` Map。在应用 AI 建议时仅对受影响人员执行增量更新，大幅减少了在 `applySuggestions` 内循环中触发全量审计 (`runAudit`) 的性能开销。
  - **O(1) 查找节假日 (O(1) Holiday Lookup)**：将 `dateUtils.ts` 中的节假日和调休日数组转换为 `Set<string>` 结构。在遍历海量日期范围时，将工作日判断的复杂度从线性查找降为常数级查找。
  - **矩阵共享与单次构建 (Single Calendar Build)**：确保 `runAudit`、日期推算及 AI 窗口生成共享同一套内存矩阵。排期过程中不再对同一个资源重复生成独立的日历副本，进一步降低了内存抖动和计算冗余。
  - **工作日预计算 (Working Day Pre-calculation)**：系统在排期开始时一次性构建排期窗口内的完整工作日 `Set`，后续所有 `getWorkingDays` 和 `calculateEndDate` 逻辑直接查表计数，彻底消除了重复的逐日逻辑判定。
- **排期准确性与健壮性提升 (Accuracy & Robustness)**：
  - **测试准入逻辑修正 (Test Entry Midpoint Logic)**：根据 PRD 规范，将测试人员的最早介入日期修正为该项目所有已分配开发任务时间跨度的**中点 (Midpoint)**，而非简单的开发开始日。这有效解决了测试资源在开发前期过早占位、导致空等的问题。
  - **回滚条件策略放宽 (Relaxed Rollback Conditions)**：增强了 PASS 2 的完整性审计逻辑。新增“开发严重欠配（已排 < 50%）且测试完全未排”的识别规则。此类项目会被果断回滚并加入重试队列，将宝贵的人力释放给后续能闭环的项目。
  - **计算精度全链路保持 (Full Precision Calculation)**：内部排期缺口 (`Gap`)、闲置天数 (`idleMd`) 及人天计算在流转过程中完全保留浮点精度，仅在写入 IndexedDB 持久化和 UI 最终渲染时执行四舍五入。有效消除了因连续取整导致的累积误差和边界值漂移。
  - **AI 结果严格校验 (Strict AI Response Validation)**：在解析 LLM 返回的 JSON 数组时，新增了严格的 Schema 过滤层。自动剔除 `projectId`/`resourceId` 缺失、分配人天小于 1 或百分比超出 (1-200%) 范围的脏数据，并记录警告日志。
  - **即时中断网络请求 (Immediate Fetch Interruption)**：在 `SchedulingContext` 中引入了 `AbortController`。现在点击「停止排期」不仅会停止逻辑循环，还会立即强制切断正在进行中的 AI HTTP 请求，确保网络资源即时释放。
- **支持「继续排期」功能 (Continue Scheduling Support)**：在全局排期大盘新增「继续排期」按钮。与「一键排期」清空重排不同，「继续排期」会保留 IndexedDB 中已有的分配数据，并在当前资源矩阵基础上增量排期，支持对新导入的项目进行追加分配或分阶段调度。
- **Token 消耗优化 (Cost Optimization)**：
  - **上下文上下文裁剪 (Context Trimming)**：对发送给 AI 的资源 `scheduleSummary` 字段执行智能裁剪。当空闲窗口信息过多导致 JSON 长度超限时，自动仅保留最近 3 个空闲区间，大幅缩减了高并发调度时的 Token 费用。
- **其他更新**：
  - **开源协议变更**：将项目的授权协议变更为 **MIT License**，鼓励社区贡献与二次开发，同时允许个人及商业用途的免费使用。

## [1.0.2] - 2026-05-03

### 优化与修复 (Improvements & Fixes)
- **持久化后台排期与手动中断**：
  - **切页不中断**：重构了调度引擎架构，将其提升至全局上下文。现在启动排期后，用户即使切换到其他管理页面，排期任务也会在后台持续进行，切换回大盘可无缝同步进度。
  - **停止排期功能**：新增了「停止排期」按钮，支持毫秒级的激进中断。点击后会立即抛出手动停止异常并切断后续 AI 调用，快速释放资源。
- **负责人强制锁定与明细匹配**：
  - 强化了 AI 调度指令，强制要求将项目定义的「技术负责人」与「质量负责人」优先排入对应项目，并确保其占有主要的分配比例，避免边缘化。
  - **业务明细联动**：AI 现在会深度解析任务明细中的产品子项（如 Order, POS, Stock 等），并自动关联人员的「业务领域能力」标签进行精准派发。
- **反碎片化排期算法优化**：在 AI 调度指令中新增了「最小分配单元」约束（强制 3 天起排，除非缺口不足）和「负责人集中化」规则。有效防止了在均衡模式下项目被过度拆解为多个 1-2 天极小碎片的现象，确保项目交付的连续性。
- **人员列表导出**：在人员管理页面新增了「人员导出」功能，支持将当前库中的所有团队成员及其角色、负荷、技能标签一键下载为 CSV 文件，方便备份与跨环境同步。
- **角色与技能联动优化**：精简了人员角色（移除了 APP 工程师），并实现了核心技能与「技能管理」页面的标签联动，支持通过点击标签快速指派能力。
- **技能标签管理页面**：新增独立的技能管理入口，支持对「业务领域能力」（更新默认值）与「技术能力」（更新默认值）进行标准化的 CRUD 管理。支持技能标签的批量导入、导出及模板下载功能。
- **人员管理批量导入**：在人员管理页面新增了上传 CSV/Excel 功能及下载导入模板功能。支持通过表头模糊匹配快速批量录入团队成员角色、负荷与技能标签。
- **已排项目概览看板**：在大盘详情上方新增「已排项目」区块，清晰列出已完成开发/测试双向资源匹配的项目，并展示技术负责人、质量负责人及投入人员明细。
- **默认策略调整**：排期大盘默认选择「均衡模式 (Balanced)」，鼓励分时并行以提升资源利用率。
- **调试能力增强**：在浏览器控制台新增详细的 [AI Debug] 和 [Hard Logic] 日志，完整追踪从 AI 建议到 JS 硬扣减的决策全路径。
- **版本迭代**：正式开启 1.0.2 版本的开发与演进。

## [1.0.1] - 2026-05-02

### 新特性 (New Features)
- **像素级时间槽位矩阵统筹 (Time Slot Matrix Scheduling)**：底层重构为基于「每日可用百分比」的日历矩阵。系统现在能感知资源在日历上的像素级分布，真正实现了 50%+50% 的多项目完美并行。
- **AI 窗口化感知与贪心收敛循环 (Convergence Loops)**：传给 AI 精确的 `Available Windows` 阵列，并引入最多 3 轮的收割循环。系统会持续尝试分配直到资源彻底收敛，资源利用率提升至 90% 以上。
- **回滚重试队列机制 (Integrity Retry Queue)**：被审计回滚的项目会自动进入重试队列，在全局收割阶段获得最高优先级，确保高优项目的残余缺口能被优先补齐。
- **AI 优先级微批次排期与完整性回滚 (Priority Mini-Batches & Integrity Rollback)**：
  - 弃用容易产生“先到先得”次优解的一把梭分配。现在系统严格按照 CSV 导入的高低优先级，将项目划分为“小批次”发送给大模型处理。
  - **排期完整性审计 (Integrity Audit)**：在所有批次排期完成后，系统会执行事务级检查，自动剥离并回滚只排上开发没排上测试的“半拉子工程”，将闲置资源退回。
- **AI 批量全局排期与 Prompt Caching (Global Batch Scheduling)**：通过合并单次调用的上下文并依托大模型的 Prompt Caching 机制，为批量分配节省了 90% 以上的 API Token 消耗。
- **多阶段迭代排期架构 (Two-Phase Scheduling)**：将排期阶段分离为 Dev-first 和 Test-second，优先安排开发任务。
- **测试前置依赖双重约束**：动态计算项目所有开发任务的起止时间中点作为测试最早开始时间，同时在 AI Prompt 中加入强制校验规则（确保测试不可早于开发），防止测试人员被盲目锁定。
- **AI 排期准则完全可配置 (Editable AI Prompt)**：在“系统设置”页面提供了一个可编辑的多行文本框，暴露了 AI 排期引擎的核心调度 Prompt。允许高级用户直接向大模型追加自定义的排期准则与限制，并提供了一键恢复系统默认规则的按钮。
- **排期策略动态切换 (Strategy Templates)**：新增“专注模式(100%投入)”、“均衡模式(50%分时并行)”与“紧急模式(超负荷推进)”等三种排期策略。
- **节假日与日历可配置 (Configurable Holiday Calendar)**：新增系统“节假日管理”页面，支持自定义调整全局法定假日和调休工作日，避免跨年问题。
- **严格排期时间窗口与 Token 优化 (Strict Window & Early Exit)**：
  - 系统强制以用户选择的结束月份为绝对物理边界（如选 4-6月则 6月30日为死线），超过此边界的项目需求将被截断为未排缺口。
  - **引擎熔断优化**：若某位工程师档期已被排满或越界，AI 引擎会自动停止在后续项目中询问该工程师；当所有工程师资源均被分配完毕时，系统将直接触发早期退出（Early Exit）中止后续循环请求，大幅节省 API Token 和执行时间。
- **一键下载导入模板 (Sample CSV Download)**：在项目管理页面提供带有标准多维表头及脱敏示例数据的 CSV 模板文件下载。
- **排期进度与错误反馈增强**：
  - 增加分布状态面板，实时反馈「初稿->审计->补排」的流转状态。
  - 优化错误弹窗提示（AI Key 过期、网络报错等）及一键跳转配置功能。

### 优化与修复 (Improvements & Fixes)
- **杜绝“0天排期”问题**：底层强依赖代码进行 `Math.min` 硬扣减截断计算，并增设严格的 $\ge$ 1 整数人天过滤逻辑。
- **重构全栈工程师职责**：将“全栈工程师”的默认匹配严格限定在开发（Dev）阶段（禁止其代替测试），以保证核心测试人员利用率。
- **增强文件导入兼容性 (Enhanced File Import)**：改用智能表头模糊匹配替代按列号读取，支持识别 `Tech Lead`、`Quality Lead`、`Details Product MD`、`Tech Stack` 及 `Domain` 等多个扩展业务字段，并在数据大盘中结构化展示。
- **项目过滤与分离展示**：大盘中自动隔离 `devTotalMd` 和 `testTotalMd` 均为 0 的项目，放置在底部“待评估项目”清单中，不参与 AI 排期。
- **已排期任务分组视图**：大盘支持「按人员分组」与「按项目分组」双向切换，方便多维统计查看。
- **人员管理体验优化**：修复人员负荷滑动条（Range Input）因步长限制无法精确设置为 0% 或 100% 的边界值问题。
- **时间合法性容错校验**：增强了日期有效性校验，修复了因导入脏数据导致的 `RangeError: Invalid time value`。
- **精简 Chrome 权限**：移除了未使用的 `alarms` 权限，以满足 Chrome Web Store 审核的安全规范。
- **完善 Chrome Web Store 上架素材**：生成了全新的 SVG & PNG (`16/48/128px`) 品牌 Logo 并引入自动化打包与发布命令 (`npm run publish`)。

## [1.0.0] - 2026-04-26

### 初版发布 (Initial Release)
- **核心排期引擎上线**：发布基于纯客户端 (Local-first) 架构的 Intelligent Resource Planner (IRP) 首个正式版本，所有数据缓存在浏览器本地 IndexedDB，彻底保障数据隐私。
- **项目数据接入**：
  - 支持通过上传 CSV 或 Excel (.xlsx) 文件，一键批量导入待排期的项目清单、优先级和预估的研发/测试工时。
  - 项目优先级严格遵循导入文件从上到下的物理顺序，确保符合现实管理直觉。
- **人员与资源管理**：
  - 提供可视化的人员管理面板，支持录入团队成员的专业角色（前端/后端/全栈/APP/测试）、当前可用负荷比例（1-100%）及各类技术栈技能标签。
- **AI 智能排期大盘 (Dashboard)**：
  - 接入大语言模型 (支持 OpenAI 及各类兼容的本地/云端 LLM 接口)，基于全局待排项目和可用资源池一键生成智能排班建议。
  - 独创的双向核算审计看板：直观展示每个项目的剩余人天需求（资源缺口），以及每个团队成员的当前闲置时长（空置资源）。
  - 根据分配占比和项目周期，自动推算跨月份的有效投入人天（MD）。
- **Jira 深度集成与原生级预警 (Content Script)**：
  - 插件支持在后台异步关联 Jira 平台的工作动态。
  - 当项目经理浏览 Jira 的 Issue 页面时，系统会实时读取该 Issue 负责人的本地排期库负荷，并在页面 DOM 内无侵入地悬浮注入其“红黄绿”过载状态，提供风险干预预警。
- **系统底层与合规**：
  - 完整的选项配置页面，支持自定义配置大模型 API 域名、密钥与指定模型。敏感密钥使用 Chrome Storage 隔离加密存储。
  - 标准化开发文档、打包流程、隐私合规声明 (Privacy Policy) 以及 SVG 品牌规范已初步建立。
